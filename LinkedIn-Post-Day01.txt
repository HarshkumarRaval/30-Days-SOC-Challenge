Day 1 of 30 ✅ — and rule #1 of building a SOC: design before you deploy. 🛡️

Today I mapped the entire lab as a logical network diagram in draw.io — every server, every data flow, before spinning up a single VM.

The build (all on Vultr, inside one VPC — 172.31.0.0/24):
🔹 Elastic & Kibana → the SIEM core
🔹 Windows Server (RDP) + Ubuntu Server (SSH) → the monitored endpoints
🔹 Fleet Server → manages the agents shipping logs
🔹 OS Ticket → case management
🔹 C2 Server (Mythic) → kept OUTSIDE the VPC as attacker infrastructure

Then I traced the flow that the whole month depends on:
endpoint → agent → Fleet → Elastic → alert → ticket

It feels almost too simple on day one. But this diagram is the mental model — every detection, every investigation later in the challenge slots into it. And honestly? It's the fastest way to explain a whole environment to an interviewer in 30 seconds.
